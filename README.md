CS 230 – Operating Platforms
Draw It or Lose It Software Design Document
Project Overview

In this project, I worked as a technology consultant for The Gaming Room, a client seeking to expand their existing Android-based game, Draw It or Lose It, into a scalable web-based application. The client required a distributed, multi-user system capable of running across multiple operating platforms while supporting multiple concurrent game sessions.

The primary goal was to design a system architecture that addressed scalability, storage management, memory management, distributed communication, and security requirements.

What I Did Well

One of my strongest contributions in this project was clearly analyzing and comparing traditional operating systems with cloud-based platforms. I effectively evaluated Linux, Windows, Mac, and mobile environments and provided a justified recommendation for a Linux-based cloud deployment.

I also demonstrated strong understanding of distributed architectures, memory considerations for high-resolution image rendering, and secure communication across platforms.

What I Learned from the Design Process

Working through a formal software design document before implementation reinforced the importance of architectural planning. Designing first helped clarify:

How memory should be managed when rendering large image libraries

How storage should be structured for both structured game data and object-based image files

How distributed systems communicate through REST APIs

How security responsibilities differ between traditional and cloud environments

This process made the technical requirements clearer and more intentional.

What I Would Improve

If revising this project, I would expand the architecture section with additional visual diagrams and deeper cost-analysis comparisons between cloud providers. Adding infrastructure-as-code examples or deployment diagrams would further strengthen the document from a professional portfolio perspective.

Interpreting User Needs

I translated the client’s requirements into technical design decisions by focusing on:

Ensuring unique identifiers and game sessions

Supporting concurrent gameplay

Managing high-definition image storage efficiently

Designing for cross-platform compatibility

Considering user needs is critical because system architecture directly impacts performance, scalability, and long-term maintainability. A well-designed system supports both business goals and user experience.

Future Design Approach

In future projects, I will continue using:

Structured design documentation

Architectural comparison analysis

Cloud-first scalability thinking

Security-by-design principles

Distributed system modeling

This project strengthened my understanding of how operating systems, cloud infrastructure, and application design intersect in real-world development environments.
